# Demo Video Script — SafeConnect (3 minutes)

00:00–00:10 — Opening (10s)
- Shot: App splash + title text "SafeConnect — Offline Disaster Communication"
- Voiceover: "When networks fail, people still need to connect."

00:10–00:30 — Show setup & discover (20s)
- Show app requesting location; user accepts.
- App scans for peers on LAN; two devices appear.
- VO: "SafeConnect discovers nearby devices and forms a peer mesh."

00:30–01:00 — Quick status action (30s)
- User A presses "Need Help". Animated pin appears on map (red).
- User B receives alert and sees pin light up; opens chat with details.
- VO: "Quick-action buttons broadcast urgent status with location."

01:00–01:40 — Messaging (40s)
- Show user-to-user message via P2P datachannel.
- IndexedDB persistence: reload the page and show messages still present.
- VO: "Messages are exchanged directly between devices and stored locally."

01:40–02:20 — Map & context (40s)
- Map view showing multiple pins with statuses.
- Tap a pin to open details — last message, timestamp, and action buttons.
- VO: "The map helps responders find users in need visually."

02:20–02:50 — Optional signaling fallback (30s)
- Explain: "If NAT prevents direct discovery, our optional lightweight signaling server helps establish connections."
- Show a quick schematic or small terminal where the signaling server runs.

02:50–03:00 — Close (10s)
- Closing shot: app logo and tagline: "SafeConnect — when networks fall, humans connect."

Notes:
- Keep the demo hands-on: show real-time interactions.
- Emphasize offline-first, privacy, and simplicity.
